# DBMS Master Tutorial Guide

Welcome to the **Database Management System (DBMS)** Master Guide! This folder contains everything you need to master DBMS concepts, specifically designed with **real-world analogies and scenarios** to help you naturally memorize the complex theories without rote learning.

## 🚀 How This Guide Works
Instead of boring textbook definitions, every concept here is explained using relatable scenarios: 
- **ER Models** are explained like drafting an architectural blueprint for a university.
- **Normalization** is explained like Marie Kondo organizing a messy closet.
- **Transactions (ACID)** are explained using real Bank Transfers.
- **Indexing (B-Trees)** is explained like the index of a 1,000-page encyclopedia.

## 📂 Folder Structure
We have broken down the course into 7 comprehensive modules:
- `01_Introduction.md`: DBMS Basics & Architecture (1-tier, 2-tier, 3-tier)
- `02_ER_Model.md`: Entity-Relationship Diagrams & Mapping
- `03_Relational_Model.md`: Keys, Dependencies, and Anomalies
- `04_Normalization.md`: 1NF, 2NF, 3NF, BCNF & Redundancy
- `05_Relational_Algebra.md`: Mathematical operators of SQL
- `06_Transactions.md`: ACID Properties, Concurrency, & Locking
- `07_Indexing.md`: B-Trees, B+ Trees, and File Organization

Let's dive in and start with `01_Introduction.md`!
